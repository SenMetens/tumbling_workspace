> 버전 1.0 · 2026-07-07 · 기획서 v1.0 기준

## 1. 기술 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | React 18 + TypeScript + Vite | 컴포넌트 재사용, 타입 안정성, 빠른 빌드 |
| 라우팅 | React Router v6 | 표준, 라우트 단위 lazy 로딩 |
| 서버 상태 | TanStack Query v5 | Supabase 쿼리 캐싱·낙관적 업데이트 |
| 스타일 | Tailwind CSS | 레퍼런스의 유틸리티성 디자인에 적합, 반응형 용이 |
| 드래그앤드롭 | @dnd-kit | 칸반, 터치 지원 |
| 마크다운 | react-markdown + CodeMirror 6 | 문서 에디터 |
| 코드 강조 | CodeMirror 6 (스니펫 편집) + Shiki 또는 highlight.js (읽기) | 문법 강조 |
| 날짜 | date-fns | 경량 |
| 백엔드 | Supabase (PostgreSQL, Auth, Realtime) | 사용자 확정 |

## 3. 데이터베이스 스키마 (Supabase / PostgreSQL)

단일 워크스페이스로 시작하되 모든 테이블에 `workspace_id`를 두어 멀티 워크스페이스 확장 여지 확보.

```sql
-- 워크스페이스
create table workspaces (
  id          uuid primary key default gen_random_uuid(),
  name        text not null,
  created_at  timestamptz not null default now()
);

-- 프로필 (auth.users 1:1)
create table profiles (
  id          uuid primary key references auth.users on delete cascade,
  name        text not null,
  job_title   text,
  avatar_url  text,
  created_at  timestamptz not null default now()
);

-- 멤버십 (워크스페이스 ↔ 사용자, 역할)
create table memberships (
  workspace_id uuid not null references workspaces on delete cascade,
  user_id      uuid not null references profiles on delete cascade,
  role         text not null default 'member'
               check (role in ('owner','admin','member')),
  created_at   timestamptz not null default now(),
  primary key (workspace_id, user_id)
);

-- 초대
create table invitations (
  id           uuid primary key default gen_random_uuid(),
  workspace_id uuid not null references workspaces on delete cascade,
  email        text not null,
  role         text not null default 'member',
  invited_by   uuid references profiles,
  accepted_at  timestamptz,
  created_at   timestamptz not null default now()
);

-- 프로젝트
create table projects (
  id           uuid primary key default gen_random_uuid(),
  workspace_id uuid not null references workspaces on delete cascade,
  name         text not null,
  description  text,                     -- 마크다운 (scope/outcome 포함)
  status       text not null default 'active'
               check (status in ('backlog','active','done','archived')),
  priority     text check (priority in ('low','medium','high')),
  label        text,
  due_date     date,
  lead_id      uuid references profiles, -- PIC
  created_at   timestamptz not null default now()
);

-- 보드 컬럼 (프로젝트별 커스텀 가능)
create table board_columns (
  id          uuid primary key default gen_random_uuid(),
  project_id  uuid not null references projects on delete cascade,
  name        text not null,
  position    int  not null,
  is_done     boolean not null default false  -- 진행률 계산용
);

-- 태스크
create table tasks (
  id           uuid primary key default gen_random_uuid(),
  workspace_id uuid not null references workspaces on delete cascade,
  project_id   uuid references projects on delete cascade,
  column_id    uuid references board_columns on delete set null,
  task_no      int  generated always as identity,   -- TSK-42 표기용
  title        text not null,
  description  text,
  priority     text check (priority in ('low','medium','high')),
  due_date     date,
  assignee_id  uuid references profiles,
  position     numeric not null default 0,          -- 컬럼 내 정렬 (fractional indexing)
  completed_at timestamptz,
  created_by   uuid references profiles,
  created_at   timestamptz not null default now()
);

-- 서브태스크
create table subtasks (
  id        uuid primary key default gen_random_uuid(),
  task_id   uuid not null references tasks on delete cascade,
  title     text not null,
  done      boolean not null default false,
  position  int not null default 0
);

-- 코멘트
create table comments (
  id         uuid primary key default gen_random_uuid(),
  task_id    uuid not null references tasks on delete cascade,
  author_id  uuid references profiles,
  body       text not null,
  created_at timestamptz not null default now()
);

-- 일정
create table events (
  id           uuid primary key default gen_random_uuid(),
  workspace_id uuid not null references workspaces on delete cascade,
  project_id   uuid references projects on delete set null,
  title        text not null,
  starts_at    timestamptz not null,
  ends_at      timestamptz not null,
  all_day      boolean not null default false,
  color        text,
  created_by   uuid references profiles,
  created_at   timestamptz not null default now()
);

create table event_attendees (
  event_id uuid references events on delete cascade,
  user_id  uuid references profiles on delete cascade,
  primary key (event_id, user_id)
);

-- 문서
create table documents (
  id           uuid primary key default gen_random_uuid(),
  workspace_id uuid not null references workspaces on delete cascade,
  project_id   uuid references projects on delete set null,
  title        text not null default '제목 없음',
  body         text not null default '',   -- 마크다운
  tags         text[] not null default '{}',
  created_by   uuid references profiles,
  updated_at   timestamptz not null default now(),
  created_at   timestamptz not null default now()
);

-- 코드 스니펫
create table snippets (
  id           uuid primary key default gen_random_uuid(),
  workspace_id uuid not null references workspaces on delete cascade,
  title        text not null,
  language     text not null default 'plaintext',
  code         text not null default '',
  description  text,
  tags         text[] not null default '{}',
  created_by   uuid references profiles,
  updated_at   timestamptz not null default now(),
  created_at   timestamptz not null default now()
);

-- 인덱스 (주요 조회 경로)
create index on tasks (workspace_id, assignee_id);
create index on tasks (project_id, column_id, position);
create index on events (workspace_id, starts_at);
create index on documents using gin (tags);
create index on snippets using gin (tags);
```

### 3.1 RLS 정책 (요지)

모든 테이블 RLS 활성화. 핵심 헬퍼:

```sql
create function is_member(ws uuid) returns boolean
language sql stable security definer as $$
  select exists (
    select 1 from memberships
    where workspace_id = ws and user_id = auth.uid()
  );
$$;
```

| 테이블 | select | insert/update | delete |
|---|---|---|---|
| workspace 소속 데이터 전체 | `is_member(workspace_id)` | `is_member(workspace_id)` | 작성자 본인 또는 admin/owner |
| memberships / invitations | 멤버 | admin/owner만 | admin/owner만 |
| profiles | 같은 워크스페이스 멤버 | 본인만 | — |

`board_columns`, `subtasks`, `comments`, `event_attendees`는 부모 행 join으로 workspace 소속 검사.

### 3.2 Realtime

`tasks`, `board_columns`, `comments`에 Realtime 구독 → TanStack Query 캐시 무효화로 반영.
낙관적 업데이트: 드래그 직후 로컬 캐시 먼저 변경, 실패 시 롤백.

## 4. 인증 흐름

1. 이메일+비밀번호 가입/로그인 (Supabase Auth)
2. 가입 트리거로 `profiles` 행 자동 생성
3. 초대: admin이 `invitations` 생성 → 초대 링크(`/login?invite=<id>`) 공유 → 가입/로그인 시 이메일 일치 확인 후 `memberships` 생성
4. 미인증 라우트 접근 → `/login` 리다이렉트, 워크스페이스 미소속 → 온보딩(워크스페이스 생성 or 초대 대기) 화면

## 6. 라우팅 & 코드 스플리팅

`App.tsx`에서 `<AuthGuard>` + `<AppShell>` 아래 각 페이지를 `React.lazy`로 로드.
기획서 3장 화면 목록과 1:1 대응. 태스크 상세는 라우트가 아닌 쿼리 파라미터(`?task=<id>`)로 열어 보드 컨텍스트 유지(PC 사이드 패널 / 모바일 풀스크린 시트).

## 8. 리스크 및 가정

- **가정**: 코드 관리 = 스니펫 (기획서 2.6). Git 연동이면 M4 재설계 필요
