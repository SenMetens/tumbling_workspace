# 워크스페이스 기획서

> 버전 1.0 · 2026-07-07 · 상태: 초안 (검토 대기)

## 1. 개요

### 1.1 목적
소규모 팀이 프로젝트·태스크·일정·문서·코드 스니펫을 한곳에서 관리하는 웹 워크스페이스.
PC와 모바일 브라우저 모두에서 사용 가능한 반응형 웹앱으로 개발한다.

### 1.2 대상 사용자
- 소규모 팀 (약 3~15명 규모 가정 — **추정, 확인 필요**)
- 팀원 전원이 동일 권한으로 협업하되, 워크스페이스 관리자(Owner/Admin) 역할 구분

### 1.3 결정 사항 (사용자 확정)
| 항목 | 결정 |
|---|---|
| 핵심 모듈 | 프로젝트 관리 / 태스크·칸반 / 캘린더 / 문서·노트 / 코드 관리 |
| 데이터 저장 | Supabase (PostgreSQL + Auth + Realtime) |
| 기술 스택 | Claude 제안 → 설계서 2장 참조 |
| 사용 형태 | 팀용 (소규모) |
| 지원 환경 | PC + 모바일 브라우저 (반응형) |

### 1.4 디자인 방향 (레퍼런스 분석)
`WorkSpace Sample` 폴더 이미지 12장의 공통 특성:

- **라이트 테마** — 흰색/오프화이트 배경(#F5F5F4 계열), 카드는 순백 + 얇은 보더 + 미세한 그림자
- **좌측 사이드바 내비게이션** — 워크스페이스 로고/전환 → 검색 → 메뉴 그룹(Essentials / Projects / Support) → 하단 설정, 각 메뉴에 카운트 뱃지
- **콘텐츠 영역** — 큰 페이지 타이틀 + 우측 상단 주요 액션 버튼(짙은 색 pill), 그 아래 탭 (Overview / Lists / Board / Timeline 등)
- **카드·테이블 혼용** — 멤버는 카드 그리드, 태스크는 행 리스트 또는 칸반 카드, 데이터는 테이블
- **상태 표현** — 컬러 pill 뱃지 (Active=초록, High=빨강, Medium=노랑, In Progress=파랑 등)
- **아바타 스택**, 둥근 모서리(8~16px), 넉넉한 여백, 미니멀 아이콘(라인형)
- 포인트 컬러는 절제 — 대부분 무채색, 상태/우선순위에만 컬러 사용

## 2. 핵심 모듈 정의

### 2.1 대시보드 (홈)
모든 모듈의 요약 진입점.
- 내 태스크 요약 (오늘 마감, 진행 중, 지연)
- 진행 중 프로젝트 카드 (진행률 바)
- 오늘/이번 주 일정 미리보기
- 최근 문서·코드 스니펫

### 2.2 프로젝트 관리 (레퍼런스: 샘플 15번)
- 프로젝트 목록: 카드 그리드(PC) / 리스트(모바일), 상태·진행률·멤버 표시
- 프로젝트 상세: 설명(In scope / Out of scope / Expected Outcomes 구조), 우측 속성 패널(Status, Priority, Due date, 담당자, 라벨), 소속 태스크 목록, 첨부 링크
- 진행률은 소속 태스크 완료 비율로 자동 계산
- CRUD: 생성/수정/보관(아카이브)/삭제

### 2.3 태스크 / 칸반 보드 (레퍼런스: 샘플 13, 16번)
- 뷰 전환: **리스트 뷰** ↔ **보드(칸반) 뷰** (탭)
- 칸반 컬럼: To Do / In Progress / In Review / Done (컬럼 추가·이름 변경 가능)
- 태스크 카드: 태스크 키(예: `TSK-42`), 제목, 설명 요약, 마감일, 우선순위 pill(High/Medium/Low), 담당자 아바타, 코멘트 수
- 드래그앤드롭으로 상태 변경·순서 변경 (모바일: 터치 드래그 또는 상태 변경 메뉴)
- 필터: 프로젝트, 담당자, 우선순위, 마감일 / 검색
- 태스크 상세: 사이드 패널(PC) / 전체 화면(모바일) — 설명, 서브태스크 체크리스트, 코멘트
- "내 태스크" 전용 화면 (전체 프로젝트 횡단)

### 2.4 캘린더 / 일정
- 월간(기본)·주간 뷰, 모바일은 월간 + 선택일 일정 리스트
- 일정 CRUD: 제목, 기간(종일/시간), 참석자, 프로젝트 연결(선택), 색상
- 마감일 있는 태스크를 캘린더에 자동 표시 (토글 가능)

### 2.5 문서 / 노트
- 폴더 없이 플랫 리스트 + 태그 분류 (규모가 작으므로 단순하게 시작)
- 마크다운 기반 에디터 (제목, 본문, 태그), 자동 저장
- 프로젝트 연결(선택) — 프로젝트 상세에서 연결 문서 노출
- 검색: 제목·본문 전문 검색

### 2.6 코드 관리 (레퍼런스: 샘플 Query Tool, ERD)
> **가정:** Git 저장소 호스팅이 아니라 **팀 코드 스니펫 저장·공유** 기능으로 정의함. (SQL 쿼리, 유틸 함수, 설정 파일 등을 문법 강조와 함께 저장·검색) Git 연동이 필요하면 범위 재협의.
- 스니펫: 제목, 언어 선택(문법 강조), 코드 본문, 설명, 태그
- 좌측 트리/리스트에서 스니펫 탐색 (레퍼런스 Query Tool 레이아웃)
- 원클릭 복사, 언어·태그 필터, 검색

### 2.7 팀 / 멤버 (레퍼런스: 샘플 12번)
- 멤버 카드 그리드: 아바타, 이름, 직함, 이메일, 상태 뱃지
- 이메일 초대(Supabase Auth), 역할: Owner / Admin / Member
- 프로필: 이름, 직함, 아바타 이미지

## 4. 반응형 원칙

| 구간 | 폭 | 레이아웃 |
|---|---|---|
| 모바일 | < 768px | 사이드바 → 하단 탭바(홈·태스크·캘린더·더보기) + 햄버거 드로어, 카드 1열, 상세는 전체 화면 |
| 태블릿 | 768–1024px | 접힌 사이드바(아이콘만), 카드 2열 |
| PC | > 1024px | 풀 사이드바(240~260px), 카드 3~4열, 상세는 사이드 패널 |

- 터치 타깃 최소 44px, 칸반 모바일 드래그는 롱프레스
- 모든 핵심 기능이 모바일에서도 동일하게 동작해야 함 (뷰만 다름)

## 5. 비기능 요구사항

- **실시간 동기화**: 태스크·칸반 변경은 Supabase Realtime으로 팀원 화면에 즉시 반영
- **권한**: 워크스페이스 단위 데이터 격리 (Supabase RLS), Owner/Admin만 멤버 관리·삭제류 수행
- **성능**: 초기 로드 3초 이내(3G Fast 기준), 라우트 단위 코드 스플리팅
- **브라우저**: 최신 Chrome / Safari / Edge, iOS Safari, Android Chrome
- **언어**: UI 한국어 (데이터는 자유 입력)

## 6. 범위 제외 (v1)

- 이메일 클라이언트, 자동화(Automations), 외부 통합(레퍼런스에 있으나 v1 제외)
- 간트 차트 (v2 후보), 파일 스토리지(문서 첨부는 v2)
- 알림(푸시/이메일) — v1은 인앱 표시만
- 다크 모드 (v2 후보)

## 7. 미확정 / 확인 필요

1. 팀 인원 규모와 워크스페이스 개수(단일 팀 전용인지, 멀티 워크스페이스인지) — **설계는 단일 워크스페이스로 시작하되 스키마는 확장 가능하게 잡음**
2. "코드 관리"의 범위 — 스니펫 관리로 가정 (2.6 참조)
3. 배포 위치 (Vercel/Netlify 등) — 설계서에서 Vercel 권장
